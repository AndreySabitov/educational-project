### Агрегация API
Для реализации эндпоинта /api/book-details/{id}, который делает 2 параллельных запроса к Book-service и Author-service
я бы выполнил следующие шаги:
1. Создал бы фильтр, имплементирующий GatewayFilter, который в своем методе использует WebClient и выполняет 2 
параллельных запроса к Book-service и Author-service. Затем с помощью Mono.zip, который дожидается выполнения обоих 
запросов, объединил бы оба результата и вернул единый ответ.
2. Через файл конфигурации создал бы роут, который обрабатывает этот запрос и использует созданный фильтр для обработки
