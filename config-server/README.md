### Config Server

Управление конфигурацией приложений имеет решающее значение для микросервисов, работающих в облаке, потому что 
экземпляры микросервисов должны запускаться быстро и с минимальным вмешательством со стороны человека.

1. Конфигурация, встроенная в JAR:

    При этом подходе все конфигурационные параметры компилируются непосредственно в исполняемый JAR‑файл приложения.

    Плюсы:
   * Простота развертывания
   * Не требуется дополнительный сервер для получения конфигураций
   * Нет задержки при старте, которая возникает при необходимости получить настройки из сервера конфигураций
   * Нет дополнительной точки отказа: конфигурация всегда доступна

    Минусы:
   * Для обновления конфигураций необходимо пересобирать и перезапустить сервис
   * Дублирование одинаковых настроек для каждого сервиса

2. Конфигурация с Config Server

   Config Server (например, Spring Cloud Config) — это отдельный сервис, который хранит конфигурации для всех приложений
и предоставляет их по запросу при старте или даже во время работы.
    
    Плюсы: 
    * Конфигурацию можно менять без перезапуска приложений
    * Все настройки в одном месте - легко вносить массовые изменения
    * Версионирование - в любой момент можно откатить настройки, при необходимости
   
    Минусы:
    * Дополнительная точка отказа: сервисы не смогут получить настройки, если config server будет не доступен
    * Задержка при старте сервисов, которая нужна для получения конфигураций
    * Зависимость от сети: из-за проблем с сетью процесс получения конфигурации может быть нарушен

### Интеграция Spring Cloud Config с HashiCorp Vault:

HashiCorp Vault - это инструмент, который управляет секретными данными. Под управлением подразумевается то, что Vault
контролирует все аспекты работы с конфиденциальной информацией: её создание, хранение, использование и отзыв.

Для интеграции Vault и Spring Cloud Config необходимо выполнить следующие шаги:
1. Запустить сервер Vault (например в docker-контейнере)
2. Создать новый способ управления секретами и сами секреты с помощью пользовательского интерфейса или командной строки
3. Указать в настройках Spring Cloud Config использовать Vault в качестве репозитория

Механизм получения секретов:
1. Клиентское приложение делает запрос к серверу конфигураций для получения настроек
2. Сервер конфигураций обращается к Vault, используя специальный токен доступа, для получения необходимых секретов
3. Сервер конфигураций объединяет полученные секреты с другими конфигурациями и передаёт их клиентскому приложению
